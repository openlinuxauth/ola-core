# Trademark

OLA and Open Linux Authentication are project marks.

Code is open source under Apache-2.0. The name is not.

Forks can use the code under the license. Forks must not imply they are the upstream OLA project.

## Fork FAQ

Can I fork the code?

Yes.

Can I call my fork OLA or Open Linux Authentication?

No. Use a different project name.

Can I say my adapter or fork is compatible with OLA?

Yes, if it implements the documented protocol and does not imply upstream endorsement.

Can I use OLA in package names?

Use descriptive names like `ola-adapter-example` only for packages meant to interoperate with upstream
OLA.

Do not use the name in a way that makes your package look official.
